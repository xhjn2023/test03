# 单元测试报告转UAT用例 + 符合度用例（一体化技能）

将单元测试报告Excel文件转换为UAT测试用例和/或符合度测试用例Excel文件。

---

## 使用方式

### 方式1：一体化转换（同时生成UAT + 符合度，推荐）
```bash
python integrated_convert.py "单元测试报告文件路径"
```

示例：
```bash
python integrated_convert.py "2月部经理陪访考核单元测试报告 .xlsx"
```

不传参数则自动处理当前目录下所有 `*单元测试报告*.xlsx`：
```bash
python integrated_convert.py
```

### 方式2：单项转换UAT
```bash
python unit_test_to_uat.py "单元测试报告文件路径"
```

### 方式3：单项转换符合度
```bash
python unit_test_to_conformity.py "单元测试报告文件路径"
```

---

## 输出文件

| 类型 | 文件名规则 | 保存位置 |
|------|-----------|----------|
| UAT测试用例 | `UAT测试用例-<前缀>.xlsx` | 与源文件同目录 |
| 符合度测试用例 | `符合度测试用例-<前缀>.xlsx` | 与源文件同目录 |

前缀 = 源文件名中"单元测试报告"前面的文案（如 `2月部经理陪访考核`）

---

## 字段映射

### UAT测试用例

| UAT表头 | 来源（单元测试报告） |
|---------|---------------------|
| 功能点* | 测试项 |
| 测试案例编号* | 用例编号 |
| 测试案例名称* | 用例描述 |
| 系统角色* | 用户（固定值） |
| 测试步骤及步骤描述* | 测试步骤 |
| 预期结果* | 预期结果 |
| 测试结果（是否通过） | 是（默认） |
| 问题描述（如未通过） | 空 |
| 测试负责人 | 空（清空） |

**样式：**
- 所有单元格添加细线边框
- 第一行表头字体加粗
- 测试结果默认为"是"
- 测试负责人清空

### 符合度测试用例

#### 测试情况 Sheet（第1个sheet）

| 符合度表头 | 来源（单元测试报告） |
|-----------|---------------------|
| 需求号 | 空 |
| 需求名称 | 文件名前缀 |
| 序号 | 用例编号 |
| 功能点简述 | 测试项 |
| 主要路径 | 测试步骤 |
| 测试结果 | 执行状态 → 通过/Pass → "成功"，其他保留原值 |
| 测试时间 | 当前系统日期（如 `6月13号`） |
| 测试人 | 王嘉裕（固定） |
| 工作量 | 空 |

**样式：**
- 第1行：合并A1:I1，文案"需求符合度测试"，宋体14px加粗白色，背景 RGB(79,129,189)，行高44.2
- 第2行：表头（需求号、需求名称、序号、功能点简述、主要路径、测试结果、测试时间、测试人、工作量），宋体14px加粗白色，背景 RGB(141,180,226)，行高42
- 数据行：从第3行开始，所有单元格细线边框、居中对齐
- 列宽：30.44, 27.78, 10.78, 42.44, 60.44, 22.22, 13.22, 9.89, 21.11

#### 测试问题归纳 Sheet（第2个sheet）

| 表头 | 说明 |
|-----|------|
| 序号 | — |
| 问题描述 | — |
| 问题类型 | — |
| 备注 | — |

**样式：**
- 第1行：合并A1:D1，文案"测试问题归纳"，宋体12px加粗白色，背景 RGB(245,124,27)，行高21.95
- 第2行：表头（序号、问题描述、问题类型、备注），宋体12px加粗白色，背景 RGB(248,162,92)，行高21.95
- 所有单元格细线边框、居中对齐

---

## 注意事项

- 自动识别 sheet 名称（优先 `寿险2021智慧银保行销支持系统` / `寿险2024版银保品质管理系统`）
- 自动定位表头行（查找包含"用例编号"、"测试项"的行）
- 自动跳过空行（用例编号为空的行）
- 符合度测试的测试人固定为"王嘉裕"，测试时间取当前系统日期
- 符合度测试不依赖模板，使用固定RGB颜色值生成全新文件
